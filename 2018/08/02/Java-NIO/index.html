Java NIO 简介与使用

  1. Linux的基础学习：鸟哥的Linux私房菜
  2. Linux网络编程：Unix网络编程
  3. Java编程基础：Head First Java

以上资料可以作为工具书，不懂的地方可以翻翻。

1. Linux IO模型

LinuxIO模型主要分为了以下5种：

  • 阻塞IO模型：这种模型是文件或者网络IO编程中最基础也是最常用的模型。以socket编程为例，在用户程序中对于fd进行read/recv的时候，如果网络数据没有到达kernel buffer的时候用户进程会从running状态变为sleep(阻塞)状态，直到网络数据的到来才将进程唤醒。
  • 非阻塞IO模型：这种IO模型与前一种的区别是，当用户程序去对fd进行read/recv的时候如果kernel buffer内没有数据。系统调用会立即返回一个error，而不会去sleep。可以看到，如果采用非阻塞IO去进行读取数据的时候，我们可能会通过轮询的方式来读取，这样做是对CPU的极大浪费。
  • 多路IO复用(IO multiplexing)：Linux的系统调用提供了select(poll)/epoll系统调用来进行多路IO复用。一个成熟的网络server都会不同时只处理一个客户端得连接，而上面得两种模式，都同时只能处理一个IO请求，所以这里得多路复用IO就显得非常有用。多路复用IO其本质是在一个IO上通说复用多个IO请求。
  • 信号驱动IO：此模型需要进程主动去开启IO的信号驱动功能。当我们进行IO操作的时候，当kernel有数据的时候会发送一个SIGIO信号，进程就会调用信号处理函数进行数据的读/写。
  • 异步IO(AIO)：异步IO是linux kernel在2.6版本之后加入的功能。异步IO在读/写数据之后，立即返回成功，并且在数据处理完成之后会发送一个SIGEV_THREAD信号，此时进程会调用信号处理函数。可以看到，此模式与前面的信号驱动IO都用到了信号进行通知。但不同的是，异步IO是在数据操作完成之后进行通知，整个操作是异步的。信号驱动IO是在fd可以读/写的时候进行通知，是同步的。

2. Java NIO使用

本课程主要讲解的是Netty，而Netty内部采用的是Java NIO来进行网络IO操作，所以我们有必要先来学习一下Java的NIO。
Java NIO实在JDk1.4中引入的，在进行Java NIO实例学习之前，我们先来了解一下Java NIO中几个重要的概念。

  1. 缓冲区Buffer，缓冲区Buffer主要包含：ByteBuffer，CharBuffer，ShortBuffer等等。在众多的Buffer中最常用的就是ByteBuffer，Buffer包含了所要读取或者是写入的数据，Java NIO中的数据是直接写入缓冲区中或者从缓冲区中读出的。这与传统的Java BIO通过Stream的方式来读取和写入是不同的。
  2. NIO通道Channel，我对于Channel的理解是：在Java NIO中的channel对应于传统BIO的InputStrimg(OutputStream)，但BIO的Stream是向一个方向流动的。而Java NIO的Channel是全双工的，这与底层操作系统的socket是一致的。
  3. 选择器Selector，这里的选择器是对于“Selector”进行字面上的翻译，实际上Selector对用于Linux的select/epoll系统调用，主要的作用是在多路复用IO，所以叫多路复用器应该比较妥帖。Selector实际上就是对于注册到上面的channel就行事件监听。

先看构造函数，在构造函数中我们通过 SocketChannel.open() 函数来打开一个channel，这个channel将会是后续进行connect的channel，同时，我们还把它设置成了non-blocking。然后，我们通过 mSelector = Selector.open() 来拿到一个选择器，我们之后会把需要监听的channel都注册到这个选择器上。

之后我们在来看看 connectToServer() 函数，在此函数中我们首先通过 mSocketChannel.connect(new InetSocketAddress(mServer, mPort) 来connect到server上，由于我们之前已经将mSelector设置为non-blocking，所以这个调用时非阻塞的它立即就会返回。这个调用成功之后，我们将channel注册到selector上并注明我们关心的是read事件，然后向Server发送“Hello”字符串。但因为，TCP连接毕竟有三次握手的过程，也许网络环境可能出现问题，所以这一次connect可能会返回失败。那我们就将此channel通过 mSocketChannel.register(mSelector, SelectionKey.OP_CONNECT) 注册到selector中，并关心他的connect事件，这样当TCP连接建立起来后，我们就会收到通知。

再看看 clientStart() 函数，该函数在调用 connectToServer() 就进入while循环，在循环中我们调用select函数来阻塞监听所有注册到selector上的channel。当channel上有事件的时候，通过遍历所有的SelectionKey然后通过 handleSelect(key) 来处理。

这时候我们来看看 handleSelect 函数，进入到这个函数中证明这个SelectionKey上有事件，于是我们通过 key.isConnectable() 和 key.isReadable() 来判断到底是哪种事件。还记得吗？我们在connectToServer的时候，有可能当时是失败的，然后我们将这个channel注册到selector上了，在这里如果是connectable，则代表当前的channel是可以连接的，那么我们通过 channel.finishConnect() 函数来完成连接，之后的过程同connectToServer一样。如果，这个channel是可读的，那么说明服务器给我们发送了消息过来。那么直接调用channel的read函数来读取数据，注意这里的ByteBuffer的操作。
